# arXiv Digest Service — 要件定義書

## 1. プロジェクト概要

### 1.1 目的

arXivに毎日投稿されるAI分野の論文から、興味度の高い5本を自動選出し、日本語要約をLINEに配信するサービスを構築する。AI開発業務における日々のインプット効率化を目的とする。

### 1.2 背景

- AI開発チームへの配属に伴い、最新研究の継続的キャッチアップが必要
- arXivは毎日数百本のAI関連論文が投稿され、人手での選別は非現実的
- 既存の論文配信サービス（Papers With Code、Hugging Face Daily Papersなど）は厳選軸が固定で、自分の興味に最適化されない

### 1.3 将来構想

- 配信先を複数ユーザーに拡張し、サブスクリプション型ニュースレターとしてマネタイズ
- そのため、最初から「サービス」としての拡張性を持たせた設計とする

## 2. 機能要件

### 2.1 必須機能（Phase 1 / MVP）

|ID  |機能       |概要                             |
|----|---------|-------------------------------|
|F-01|論文取得     |arXiv APIから指定カテゴリの前日投稿論文を取得    |
|F-02|粗フィルタ    |キーワード/著者/件数で候補を200本程度に絞り込み     |
|F-03|重要度スコアリング|LLMで全候補を0-10でスコアリング            |
|F-04|Top N選出  |スコア上位5本を選出                     |
|F-05|日本語要約生成  |各論文のAbstractをLLMで日本語要約         |
|F-06|LINE配信   |LINE Messaging APIで個人へDM送信     |
|F-07|重複防止     |過去配信済み論文を記録し再送を防止              |
|F-08|API化     |HTTPエンドポイントとして全機能を提供           |
|F-09|定期実行     |Cloud Schedulerによる毎日自動実行       |
|F-10|手動実行     |スマホ等から任意タイミングで実行               |
|F-11|ドライラン    |LINE送信せず結果プレビュー                |
|F-12|コスト上限    |1日あたりのLLM API利用額に上限を設定（デフォルト$1）|

### 2.2 将来機能（Phase 2以降）

|ID  |機能          |概要                        |想定フェーズ |
|----|------------|--------------------------|-------|
|F-21|論文本文（PDF）の要約|Abstractだけでなく本文も読ませる      |Phase 2|
|F-22|精度比較機能      |複数LLMで同じ論文を要約し品質比較するAPI   |Phase 2|
|F-23|配信履歴閲覧UI    |過去の配信内容をWebで閲覧            |Phase 2|

|F-26|決済連携        |Stripe等でサブスク課金            |Phase 4|

## 3. 非機能要件

### 3.1 性能

- バッチ1回の実行時間: **10分以内**を目標、最大60分まで許容
- LINE配信: 朝7:00〜8:00（JST）に届くこと

### 3.2 コスト

- **インフラ**: 月額500円以下を目標（Cloud Run無料枠内が理想）
- **LLM API**: 1日あたり$1上限、月額$30以下を許容
- 上限超過時は当日の処理をスキップし通知

### 3.3 信頼性

- 連続失敗（3日）時は管理者にアラート
- arXiv APIまたはLLM APIの一時的障害には自動リトライで対応（最大3回、指数バックオフ）

### 3.4 セキュリティ

- API認証: 共有秘密鍵（Bearer Token）方式
- LLM APIキー・LINE APIキーは Secret Manager で管理、コードには含めない

### 3.5 保守性

- LLMプロバイダはプラグイン形式で差し替え可能（Groq / Together / OpenAI / Anthropic等）
- 興味カテゴリ・キーワード・上限値などは設定ファイルで管理

## 4. 制約事項

### 4.1 外部サービス制約

- **arXiv API**: 3秒に1リクエスト推奨（Rate Limit）
- **LINE Messaging API**: 無料枠で月200通まで、Push API利用
- **Cloud Run**: 1リクエスト最大60分

### 4.2 利用範囲

- Phase 1では配信先は開発者本人1名
- 第三者への配信は法的観点（arXivコンテンツの再配布範囲）を整理してから

## 5. ステークホルダー

- **オーナー / 開発者 / 初期ユーザー**: プロジェクト発起人
- **Claude Code**: 実装担当
- **将来の購読者**: Phase 3以降の想定ユーザー

## 6. 用語定義

|用語    |定義                       |
|------|-------------------------|
|Digest|1回の配信で送られる論文要約のまとまり      |
|粗フィルタ |LLMに渡す前のルールベースな絞り込み処理    |
|スコアリング|LLMによる重要度の数値化（0-10）      |
|ドライラン |LINE送信を行わずに処理結果のみを返す実行モード|

## 7. 成功基準

### Phase 1 完了の定義

- 毎日朝7-8時にLINEで論文5本のサマリーが届く状態が、1週間連続で稼働
- 配信内容が「読む価値あり」と感じられる品質（主観評価で5本中3本以上）
- 月額コストが目標内に収まる
- 手動実行APIがスマホから1タップで叩ける
